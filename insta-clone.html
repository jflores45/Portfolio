Instagram Clone Project

Project Overview

This project is a simplified clone of Instagram, allowing users to sign up, upload photos, follow other users, like posts, and leave comments. The goal was to replicate the core features of Instagram using modern web technologies.

Motivation/Goal

The project was inspired by the need to build a full-stack application with user authentication and CRUD operations, enhancing my skills in Flask and modern web development.

Technical Stack

Front-end: HTML, CSS, JavaScript

Back-end: Flask (Python)

Database: SQLite

APIs/Libraries: Flask-SQLAlchemy for ORM, session-based authentication

Features

  • User authentication (signup, login).
  • Upload and display photos in a user feed.
  • Follow/unfollow users and interact with posts (like, comment).
  • Responsive design for both desktop and mobile.
  • Infinite scrolling with pagination using postid_lte, size, and page query parameters for optimized data loading.

Architecture/Design

The project uses Flask for server-side operations and a REST API to handle communication between the front-end and back-end. Data is stored in SQLite, and user sessions are managed using Flask’s built-in session management to handle authentication.

The REST API supports pagination for the user feed. When fetching posts, the client can pass query parameters like postid_lte, size, and page to load a specific number of posts or continue scrolling from where the last posts were loaded.

Pagination

To support infinite scrolling, the pagination system allows the client to request posts in chunks. The query parameters include:

  • postid_lte=N: Fetch posts with an ID less than or equal to N.
  • size=N: Specify the number of posts to return per request.
  • page=N: Request a specific page of results (useful for non-infinite scrolling scenarios).

Challenges & Solutions

One challenge was ensuring user authentication was secure and efficient. Flask's session-based authentication simplified this process, ensuring only logged-in users can interact with protected routes.

Another challenge was optimizing the feed for large sets of data. Pagination using query parameters (postid_lte, size, and page) allows for efficient data retrieval and supports infinite scrolling for a seamless user experience.

Lessons Learned

This project helped me deepen my understanding of RESTful APIs, user authentication with Flask, and integrating a front-end with a back-end for a full-stack web application. I also improved my database management skills using SQLAlchemy.

Future Improvements

Future improvements include adding more user interaction features like real-time notifications for likes and follows, and implementing more efficient methods for handling larger data sets of posts.
